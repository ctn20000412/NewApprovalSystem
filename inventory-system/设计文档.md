# 企业库存管理系统 - 软件设计文档

## 1. 项目概述

### 1.1 项目名称
**企业库存管理系统（Enterprise Inventory Management System, EIMS）**

### 1.2 项目背景
本系统面向销售型企业，用于管理取货请求、订单处理、销售统计等核心业务流程。系统支持销售人员和经理两种角色，实现从需求提交、审批到最终成交的完整闭环管理。

### 1.3 技术栈
- **后端**：Java 17 + Spring Boot 3.2
- **前端**：HTML5 + Thymeleaf + Bootstrap 5
- **数据库**：MySQL 8.0
- **构建工具**：Maven
- **ORM框架**：Spring Data JPA / Hibernate

## 2. 功能需求分析

### 2.1 角色定义

| 角色 | 权限范围 |
|------|----------|
| **销售 (Sales)** | 提交取货请求、查看请求状态、提交最终售卖情况、查看个人统计 |
| **经理 (Manager)** | 审核取货请求、审批/拒绝/调整订单、查看所有统计报表、货仓管理 |

### 2.2 功能模块

#### 2.2.1 用户管理模块
- 用户登录/登出
- 用户信息维护
- 角色权限控制（销售/经理）

#### 2.2.2 产品管理模块
- 产品分类管理（类别：电子产品、办公用品、机械配件等）
- 产品信息管理（名称、规格、单价、库存数量）
- 库存查询与统计

#### 2.2.3 取货请求模块（销售操作）
- 创建取货请求：
  - 选择产品类别
  - 填写产品数量需求
  - 填写销售单价
  - 填写客户名称/项目名称
  - 填写备注说明
- 查看请求列表及状态
- 修改/撤回未处理的请求

#### 2.2.4 订单审核模块（经理操作）
- 查看待审核请求列表
- 审批操作：
  - **同意**：直接通过
  - **不同意**：填写拒绝原因
  - **调整**：修改销售价格或数量，填写调整意见
- 查看已审核订单

#### 2.2.5 订单成交模块（销售操作）
- 提交最终售卖情况：
  - 实际成交数量
  - 实际成交单价
  - 实际客户/项目
  - 成交日期
- 订单状态流转：待审核 → 审核中 → 已同意 → 已成交/已拒绝

#### 2.2.6 货仓统计模块
- 实时库存查询
- 库存变动记录
- 库存预警提示

#### 2.2.7 财务报表模块
- 订单金额统计
- 每月销售汇总
- 销售个人业绩统计
- 公司整体营收报表

## 5. 业务流程

### 5.1 取货请求流程

```
销售创建请求 → 经理审核 → [同意/拒绝/调整] → 销售确认 → 成交
```

### 5.2 状态流转

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| PENDING | 待审核 | 销售：撤回/修改；经理：审核 |
| APPROVED | 已同意 | 销售：确认成交 |
| REJECTED | 已拒绝 | 销售：查看原因 |
| ADJUSTED | 已调整 | 销售：确认调整/重新申请 |
| COMPLETED | 已成交 | 流程结束 |
| CANCELLED | 已取消 | 流程结束 |

## 7. 安全性设计

### 7.1 认证授权
- Spring Security 实现基于角色的访问控制
- 密码使用 BCrypt 加密存储
- Session 管理

### 7.2 权限控制

| URL路径 | 销售 | 经理 |
|---------|------|------|
| /dashboard | ✓ | ✓ |
| /requests/** | 自己的请求 | 所有请求 |
| /orders/** | 自己的订单 | 所有订单 |
| /products/** | 只读 | 读写 |
| /warehouse/** | ✓ | ✓ |
| /reports/** | 个人报表 | 全部报表 |

### 8.3 启动方式
- 开发环境：`mvn spring-boot:run`
