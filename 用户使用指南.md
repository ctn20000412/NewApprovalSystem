# 企业库存管理系统 · 用户使用指南

---

## 📋 系统简介

本系统是一个面向中央空调销售公司的库存管理系统，采用前后端分离架构：

| 模块 | 技术栈 | 端口 |
|------|--------|------|
| **后端** | Java Spring Boot 3.2 + MySQL 8.0 | 8080 |
| **前端** | React 18 + TypeScript + Vite 5 + Tailwind CSS | 5173 |

### 功能模块

- **用户管理**：经理、销售两种角色
- **产品管理**：产品分类、库存管理、价格管理
- **取货申请**：销售提交申请 → 经理审批 → 生成订单
- **订单管理**：订单创建、确认、完成、取消
- **库存日志**：入库、出库、库存变动记录

## 🚀 快速启动

### 方式一：一键启动（推荐）

#### 1. 初始化数据库

```bash
# 进入数据库脚本目录
cd inventory-system/database

# 登录 MySQL 并执行初始化脚本
mysql -u root -p < setup.sql

# 提示输入密码：Ning.0412
```

#### 2. 启动后端（PowerShell）

```powershell
cd inventory-system
mvn spring-boot:run
```

#### 3. 启动前端（新终端）

```powershell
cd inventory-frontend
npm install      # 首次运行需要安装依赖
npm run dev
```

**说明：**

- 前端开发服务器默认端口为 `5173`
- 前端依赖后端接口，启动前请先确认后端已经运行在 `http://localhost:8080`
- 当前前端通过 Vite 代理转发 `/api`、`/login`、`/logout` 到后端，不需要手动修改接口地址

#### 4. 访问系统

| 地址 | 说明 |
|------|------|
| http://localhost:5173 | 前端页面 |
| http://localhost:8080 | 后端 API |

## ⚙️ 配置说明

### 后端配置（application.yml）

```yaml
# 数据库连接
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/inventory_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root
    password: Ning.0412    # 修改为你的 MySQL 密码

  jpa:
    hibernate:
      ddl-auto: update     # 自动更新表结构（开发环境）

# 服务端口
server:
  port: 8080
```

## 🐛 常见问题

### 1. 后端启动报错：Lombok 编译错误

**原因**：JDK 版本与 Lombok 版本不兼容

**解决**：已配置 Lombok 1.18.36 支持 JDK 23，如仍报错：

```powershell
# 清理并重新编译
mvn clean compile -DskipTests
```

### 2. 数据库连接失败

**检查项**：
- MySQL 服务是否启动
- 数据库 `inventory_db` 是否存在
- 用户名密码是否正确

```sql
-- 检查数据库
SHOW DATABASES LIKE 'inventory%';

-- 检查用户权限
SELECT user, host FROM mysql.user;
```

### 3. 前端无法连接后端（CORS 错误）

**解决**：后端已配置 CORS 允许 `http://localhost:5173` 和 `http://127.0.0.1:5173`，如仍报错：

```java
// 检查 CorsConfig.java 配置
CorsConfiguration config = new CorsConfiguration();
config.setAllowCredentials(true);
config.addAllowedOrigin("http://localhost:5173");
config.addAllowedOrigin("http://127.0.0.1:5173");
config.addAllowedHeader("*");
config.addAllowedMethod("*");
```

### 4. 端口被占用

```powershell
# 查找占用 8080 端口的进程
netstat -ano | findstr :8080

# 结束进程（替换 PID）
taskkill /PID <PID> /F
```

### 5. 前端依赖安装或启动失败

**排查顺序：**

- 执行 `node -v`，确认 Node.js 版本不低于 20
- 执行 `npm -v`，确认 npm 可用
- 如依赖安装失败，可删除 `inventory-frontend/node_modules` 和 `inventory-frontend/package-lock.json` 后重新执行 `npm install`
- 如 `5173` 端口被占用，可执行 `npm run dev -- --port 5174`
